The Shadow DOM

The darkest object model

Shadow DOM is a feature meant to enable engineers to build black-box components with strong encapsulation.

This feature has actually been around in browsers for a while, but only now is it being exposed as an API.

As far as the rest of the page is concerned, this entire widget is just one single element.

However...

          
  // Start off by selecting the element you want to serve as the *host*.
  var hostElement = document.querySelector('#hostElement');

  // Then create a "Shadow Root" that will contain the shadowy DOM structure.
  var shadowRoot = hostElement.createShadowRoot();

  // Add some content to the shadow root.
  shadowRoot.appendChild(document.importNode(someTemplate.content));

What does being in a shadow root mean?

  • All style definitions don't cross shadow root boundaries.
  • All element IDs are scoped to their shadow root.
  • Upward node traversal will terminate at shadow root boundaries.
  • Downward node traversal will skip over shadow roots.
  • Events are retargeted when they propagate across a shadow root boundary

Event Retargeting

The target of an event is changed to the host element every time it propagates through a shadow root.

The sum of all of these features is a new tool that enables a level of encapsulation previously impossible.

Content projection

the feature formerly known as transclusion

When the template inserted into a shadow root contains a <content> element, any child DOM in the host element is projected into that <content> element.

The template element creates an inert subtree than can manually be manipulated, cloned, and inserted into the document.

This works very well with Shadow DOM, as the content of a template element can be used to clone into shadow roots.

What does this have to do with Angular?

Angular 2 embraces these new concepts at its foundation.

Components

Components in Angular 2 are heavily influced by both the structure and the encapsulation of web components.

Angular 2 applications consist of a tree of encapsulated Views. Each view contains elements than host other views. 

          
  @Component({ selector: 'cool-button' })
  @View({
    template: '

Shadow DOM Strategy

  • Native mode: actual Shadow DOM in the browser.
  • Emulated mode: Angular emulates Shadow DOM spec as closely as possible. Somewhat costly.
  • Emulated unscoped mode Emulates only content projection and event retargeting. Much less costly.

How does this affect my components?

Here are some things to keep in mind when building Angular 2 components.

The host of a component is how the rest of the world sees it for:

  • Events
  • Accessibility
  • CSS

Angular 2 uses an EventEmiter object to imperitively fire event from a component's host element. 


  @Component({
    // ...
    events: ['game-over']
  })
        

  this.gameOver = new EventEmitter();
        

  this.gameOver.next(finalScore);

Angular 2 uses a declarative syntax to set properties and attributes on the host element. 


  @Component({
    // ...
    hostProperties: {
      role: 'attr.role',
      checked: 'attr.aria-checked'
    }
  })
        

  this.checked = true;

All styles and behavioral modifiers are applied to the host element. 


          awesome-button .raised {
            box-shadow: /** ... */
          }
